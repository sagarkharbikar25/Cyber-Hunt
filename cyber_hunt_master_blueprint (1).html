🔒 All answer hashing happens server-side. API never returns the correct answer — only {correct: true/false}. Rate limit: 5 attempts/min per team.
POST/api/auth/register
Creates team row, generates TEAM-ID, sends email confirmation
Body: {team_name, leader_name, leader_email, college_name, members[]} → {team_id, message}
POST/api/auth/login
Validates email + team_id, returns HttpOnly JWT cookie (6h expiry)
Body: {email, team_id} → sets cookie, returns {team_name, current_level}
GET/api/level/[levelId]
Returns level content ONLY if team has reached this level. Server checks progress table.
Returns: {title, content_html, challenge_type} — never returns answer_hash or fragment
POST/api/level/[levelId]/submit
Hashes submitted answer server-side, compares to stored hash. Rate-limited 5 attempts/min.
Body: {answer} → {correct: bool, fragment?: string} — fragment only on correct
GET/api/hints/[levelId]
Returns available hints based on time spent on level. Logs to progress.hints_used[].
Returns: {available_hints: [{num:1, text:"..."}]} — time-unlocked only. Requires screenshot for hints 2 & 3.
GET/api/fragments
Returns all earned fragments for authenticated team from DB. Never from client state.
Returns: {fragments: [{level_id:1, value:"CY"}, ...]}
GET/api/leaderboard
Top 20 teams sorted by levels solved. Polled every 30s. Cached 25s. No Supabase Realtime.
Returns: {teams: [{rank, team_name, levels_solved}]} — score hidden during event
GET/api/dashboard
Team full state: rank, current level, fragment inventory, countdown, hint status
Returns: {team_name, rank, current_level, score, fragments[], event_status}
POST/api/submit/final
Auto-called when Level 10 solved. Creates row in final_submissions. Redirects to /submit page.
Guard: if row exists → return existing. Returns: {submission_id, solved_at, redirect:"/submit"}
POST/api/submit/upload
Upload screenshot proof. Max 5MB. One upload per team. Within event time or 10-min grace.
Returns: {success:true, uploaded_at} — stores to Supabase Storage
GET/api/results
Public but gated. Returns 403 until results_published = true in event_settings. Cached 60s.
Returns: Full leaderboard + winner flagged — only after organizer publishes
POST/api/admin/pause
Pause/resume event. Records paused_at. On resume, adds elapsed to total_paused_ms. Admin only.
Body: {action: "pause" | "resume"} — timer compensation is automatic
POST/api/admin/force-unlock
Emergency: unlocks a level for a specific team. Logged to audit trail. Admin only.
Body: {team_id, level_id}
POST/api/admin/results/publish
Sets results_published = true + winner_announced = true. Unlocks /results page for all.
Body: {winner_team_id} — admin only, run after manual verification

Elimination Rules
🚫 Two elimination types: (1) No progress timeout. (2) AI abuse via screenshot detection.
Elimination Type 1 — No Progress Timeout
10 attempts AND 0 correct submissions after 45 min → flagged as "stuck"
2Warning banner on dashboard: "No progress detected. 15 minutes before session marked inactive."
3After 60 min with 0 attempts → "eliminated" in DB. Dashboard shows "Mission Failed — Session Closed."
!Edge case: team actively trying wrong answers is NOT eliminated — attempts show genuine effort.
AI Rule — 3 uses allowed, 4th = elimination
Teams are told in pre-event briefing: AI tools are allowed up to 3 times total. This is fair — announced upfront. No penalty for first 3 detected uses.
AI Detection — Screenshot on Hint 2 or Hint 3
When a team requests Hint 2 or 3, the system shows: "To unlock this hint, upload a screenshot of your current screen." If screenshot shows AI tool UI (ChatGPT, Claude, Gemini visible) → counts as one AI-use detection. After 3 such screenshots → elimination review begins.
4th AI Detection — Disqualification
Organizer manually reviews screenshots after event. If 4+ AI-use screenshots found → team marked disqualified. Not eligible for prizes but receives participation certificate.
